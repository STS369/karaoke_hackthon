## 💡 背景・課題 (Why)
カラオケに行った際、「次、何を歌おう？」と悩み、履歴を探す時間を無駄にしてしまう経験から着想を得ました。  
**「選曲の迷い」をなくし、歌う時間を最大化する**ために、自分の聴取履歴から瞬時にセットリストを作れるアプリを開発しました。

---

## 📱 主な機能 (Features)

### 1. Spotify連携によるパーソナライズ
- OAuth認証を用いてログインし、「最近聴いた曲」「お気に入り」「おすすめ曲」を自動取得・表示

### 2. 直感的なプレイリスト作成
- 気になった曲を「リストに追加」ボタン一つでストック  
- カラオケ店での入力をスムーズにするための「セットリスト」を瞬時に構築

### 3. メモ・スコア機能
- 曲ごとに「採点結果」「歌った感想」を記録  
- 次回のカラオケに活かせるデータベースとして活用可能

## 🛠 使用技術 (Tech Stack)

### Frontend
- **Language**: JavaScript  
- **Framework**: React (Vite)  
- **Styling**: CSS（Neon UI Design）

### Backend
- **Language**: Python 3  
- **Framework**: Flask  
- **Libraries**: Spotipy（Spotify API wrapper）, Flask-CORS  
- **Authentication**: OAuth 2.0（Authorization Code Flow）

## 🧑‍💻 私の担当箇所 (Role)
**フロントエンドとバックエンドの連携（統合）担当**

本プロジェクトでは、主に **React（フロントエンド）と Flask（バックエンド）をつなぐ役割**を担当しました。  
画面側の実装を進めながら、「データがどこから来て、どこへ渡るのか」を意識して開発を行いました。

### フロントエンド × バックエンドの API 連携
- React から Flask API へリクエストを送り、結果を画面に反映する処理を実装
- JSON データの構造を確認しながら、「どの値を・いつ・どう使うのか」を整理して開発
- 通信エラーやデータ未取得時にも画面が破綻しないよう、最低限のエラーハンドリングを意識

### Git / チーム開発について
- Git は主に **自分の作業内容を正しく反映するための利用**が中心
- 大規模な管理は行っていませんが、  
  「どの変更が何のためのものか」が分かるよう、commit メッセージや変更範囲を意識して作業しました

## 🔥 こだわった点・苦労した点 (Challenges)

### 1. 実装スピードを最優先した AI 活用とリソース配分
今回、バックエンド技術（Flask）は未経験の領域でした。  
短期間のハッカソンで導入するため、**「期間内に動くものを作り上げるスピード感」**を最優先しました。

- **AIへの委任と実装への集中**  
  複雑な処理ロジックや構文の生成は AI に任せ、実装時間を短縮
- **チーム開発への貢献**  
  生成コードの入出力（インターフェース）が正しいかの確認に注力し、  
  フロントエンド連携やバグ検証に時間を割くことで、プロダクト完成に貢献

### 2. 「データは来ているのに表示されない」バグの解決
バックエンドではデータ取得できているのに、フロントエンドが真っ白になる問題が発生。  
ブラウザの検証ツールとサーバーログを突き合わせて原因を特定し、  
型定義の不整合や非同期処理のタイミングを修正して解決しました。

### 3. 「カラオケボックス」を再現した UI デザイン
ダークモード × ネオンカラーで統一し、  
「開いた瞬間にテンションが上がる」UI を目指しました。
